---
title: "API Reference — Installations"
description: "Endpoints for managing app installations and user credentials"
---

Base URL: `https://api.orceum.com/v1`

Installation endpoints require either the developer's token or the user's token depending on the operation.

---

## Install an App

```bash
POST /apps/{app_id}/install
```

Creates a new installation for the authenticated user.

**Headers:** `Authorization: Bearer USER_ORCEUM_TOKEN`

**Request body:** `{}` (empty)

**Response `201`:**

```json
{
  "installation_id": "inst_abc123",
  "app_id": "app_a1b2c3d4",
  "status": "PENDING",
  "next_action": "API_KEY",
  "created_at": "2024-01-15T12:00:00Z"
}
```

| `next_action` | Next Step |
|---------------|-----------|
| `NONE` | Installation is active immediately |
| `API_KEY` | Call `POST .../api-key` to submit credential |
| `OAUTH` | Redirect user to authorization URL |

## List User's Installations

```bash
GET /apps/{app_id}/installations
```

Returns all installations for the authenticated user for this app (typically 0 or 1).

**Headers:** `Authorization: Bearer USER_ORCEUM_TOKEN`

**Response `200`:**

```json
{
  "installations": [
    {
      "installation_id": "inst_abc123",
      "app_id": "app_a1b2c3d4",
      "status": "ACTIVE",
      "installed_at": "2024-01-15T12:00:00Z"
    }
  ]
}
```

## Get Installation Details

```bash
GET /apps/{app_id}/installations/{installation_id}
```

**Response `200`:**

```json
{
  "installation_id": "inst_abc123",
  "app_id": "app_a1b2c3d4",
  "user_id": "user_xyz789",
  "status": "ACTIVE",
  "auth_type": "API_KEY",
  "installed_at": "2024-01-15T12:00:00Z",
  "last_action_at": "2024-01-15T14:00:00Z"
}
```

## Uninstall an App

```bash
DELETE /apps/{app_id}/installations/{installation_id}
```

Removes the installation, clears credentials, calls `revoke_url` (if configured), and sends `UNINSTALLED` lifecycle webhook.

**Headers:** `Authorization: Bearer USER_ORCEUM_TOKEN`

**Response `204`: No content**

## Submit API Key

```bash
POST /apps/{app_id}/installations/{installation_id}/api-key
```

Used after install when `next_action` is `API_KEY`.

**Headers:** `Authorization: Bearer USER_ORCEUM_TOKEN`

**Request body:**

```json
{
  "api_key": "sk-user-provided-key-here"
}
```

**Response `200`:**

```json
{
  "installation_id": "inst_abc123",
  "status": "ACTIVE"
}
```

**Errors:**
- `400` — Key fails validation (too short, contains spaces, wrong format)
- `409` — Installation already has an active key

## Request Re-authentication

```bash
POST /apps/{app_id}/installations/{installation_id}/reauth
```

Generates a re-authentication URL for installations in `REAUTH_REQUIRED` state.

**Headers:** `Authorization: Bearer USER_ORCEUM_TOKEN`

**Response `200`:**

```json
{
  "reauth_url": "https://api.orceum.com/v1/apps/app_a1b2c3d4/oauth/start?state=...",
  "expires_at": "2024-01-15T13:00:00Z"
}
```

## OAuth Callback

```bash
GET /apps/{app_id}/oauth/callback
```

This endpoint is called by the OAuth provider after the user grants consent. You do **not** call this directly — register it as the redirect URI with your OAuth provider.

```
https://api.orceum.com/v1/apps/{app_id}/oauth/callback
```

Orceum handles the code exchange, token storage, and installation activation automatically.
